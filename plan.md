# План реализации MVP «Первая работа»

## Стек
- **Frontend:** текущий HTML-прототип, vanilla JS, без фреймворка
- **Backend + БД:** Supabase (PostgreSQL + Auth + REST API + Edge Functions)
- **SMS:** SMSC.ru или SMS.ru (российский провайдер)
- **Хостинг:** Vercel

## Фаза 2 — База данных: схема и таблицы

### Таблицы

- **`users`** — телефон, дата регистрации, город
- **`user_filters`** — сфера, график, зарплата (JSON)
- **`vacancies`** — id, название, компания, город, зарплата, сфера, ссылка, описание, флаг `active`
- **`sphere_tips`** — сфера, список вопросов-ответов (JSON); «что взять с собой» — универсальный блок для всех
- **`applications`** — user_id, vacancy_id, дата отклика, статус (`pending / called / silent / reject`), дата изменения статуса, раунд + поля для Excel-выгрузки (уточнить отдельно)
- **`funnel_state`** — user_id, текущее состояние воронки, target на раунд, дата последнего действия

### Дополнительно

- Уточнить набор полей `applications` для Excel-выгрузки → зафиксировать в схеме
- **Результат:** схема согласована, таблицы созданы в Supabase

## Фаза 3 — Импорт вакансий из XML фида

- Изучить структуру XML фида
- Написать скрипт-импортёр: парсит XML → кладёт вакансии в `vacancies`
- Написать тексты «что говорить» для всех 8 сфер вручную и загрузить в `sphere_tips`:
  - 🛒 Торговля
  - ☕ Кафе и рестораны
  - 💻 IT и офис
  - 📦 Склад и логистика
  - 📞 Колл-центр
  - 💇 Красота и уход
  - 👶 Образование
  - 🏥 Медицина
- Подключить реальный счётчик вакансий в фильтрах: при выборе города / сферы / графика / зарплаты — запрос `COUNT` к Supabase с текущими фильтрами; убрать захардкоженные числа и фейковую анимацию
- Подключить карточки вакансий в виджете к реальным данным из базы
- **Результат:** в виджете показываются реальные вакансии по фильтрам, счётчик отражает действительность

## Фаза 4 — Авторизация по телефону + SMS

- Вернуть экран авторизации в виджет (шаг 1: ввод телефона, шаг 2: SMS-код)
- Подключить SMS-провайдер: генерируем 4-значный OTP, отправляем SMS, храним в Supabase с TTL 5 минут
- При верном коде — создаём/находим пользователя, выдаём сессию (JWT через Supabase Auth или session token в localStorage)
- **Результат:** можно войти по телефону, после перезагрузки сессия не сбрасывается

---

## Фаза 5 — Сохранение состояния в базу

- Сохранять фильтры после прохождения шагов 1–4
- Сохранять каждый отклик в `applications` при нажатии «Откликнуться»
- Сохранять статусы при обновлении (День 2)
- При открытии виджета — восстанавливать состояние воронки из базы
- **Результат:** закрыл и открыл виджет → видишь своё состояние, не начинаешь заново

---

## Фаза 6 — SMS-уведомления (напоминания)

- Написать Edge Function в Supabase: раз в день (~9:00 мск) проверяет `funnel_state` → кому давно не напоминали → шлёт SMS
- Сценарии SMS:
  - После Дня 1: «Первая работа: пора проверить, кто ответил. Открой виджет → [ссылка]»
  - Если не возвращался 2+ дня: повторное напоминание
- Настроить Supabase Cron для ежедневного запуска
- **Результат:** тестовый пользователь получает SMS-напоминание
